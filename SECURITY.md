# 安全政策

## 支持的版本

我们只为最新版本的代码提供安全更新。

## 报告安全漏洞

如果你发现安全漏洞，请不要公开报告。请通过以下方式私下报告：

1. 发送邮件到：security@yourdomain.com
2. 使用 GitHub 的私有安全报告功能

请尽可能包含以下信息：

- 漏洞描述
- 影响的版本
- 复现步骤
- 潜在影响
- 建议的修复方案（如果有）

## 响应时间

我们会在收到报告后的 48 小时内回复，并在 7 个工作日内提供修复方案或时间表。

## 安全最佳实践

### 对于用户

1. **保护 API 密钥**：
   - 不要在代码中硬编码 API 密钥
   - 使用环境变量存储敏感信息
   - 定期轮换 API 密钥

2. **使用 HTTPS**：
   - 始终使用 HTTPS 连接
   - 验证 SSL 证书

3. **限制访问**：
   - 使用防火墙限制访问
   - 配置适当的认证
   - 使用反向代理

4. **定期更新**：
   - 及时更新到最新版本
   - 关注安全公告

### 对于开发者

1. **依赖管理**：
   ```bash
   npm audit
   npm update
   ```

2. **代码审查**：
   - 所有代码变更需要经过审查
   - 使用静态分析工具

3. **测试**：
   - 编写安全测试
   - 进行渗透测试

4. **文档**：
   - 记录安全相关的配置
   - 提供安全最佳实践指南
